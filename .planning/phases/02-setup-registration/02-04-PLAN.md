## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Imported JSON file → dexie-export-import → IndexedDB | The single highest-risk boundary in Phase 2: a file selected by the trainer is parsed and written directly into IndexedDB via a third-party library, bypassing the app's own form-level validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-02-08 | Tampering | Imported JSON file (`importDB`) can write arbitrary/malformed records directly into IndexedDB, bypassing ClassForm/PresetSave-level validation (e.g. name length, tuple shape) | mitigate | Wrap `importDB()` in try/catch; after import, validate every `db.presets` record has `name`/`classes` (array)/`shootingLineCount` (number)/`roundsConfig` (object), `bulkDelete` any that don't; surface `strings.presets.importError` on failure |
| T-02-09 | Denial of Service | Oversized or malicious import file (e.g. thousands of fake presets) | mitigate | After a successful import, trim to the first 8 valid presets by `createdAt`, deleting the rest, preventing unbounded IndexedDB growth or a preset-list render freeze |
| T-02-SC | Tampering | npm install of `dexie-export-import` | accept | Already vetted `[OK]` in 02-RESEARCH.md's Package Legitimacy Audit (official Dexie addon, 2+ years, 100K+/week downloads) — no additional checkpoint required |
</threat_model>
