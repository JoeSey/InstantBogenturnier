- **Format:** A4 portrait
- **Margins:** 20mm all sides
- **Units:** millimeters (jsPDF config: `unit: 'mm'`)

   - Font size: 10pt
   - Font: Helvetica (standard PDF font)
   - Weight: Regular
   - Color: Black
   - Position: Top-left, 20mm from left edge
2. **Class heading**
   - Font size: 14pt
   - Font: Helvetica
   - Weight: Bold
   - Color: Black
   - Position: Below title, 20mm from left edge
3. **Header images** (if configured)
   - Left logo: 25mm width, 20mm height, positioned 20mm from left edge, 5mm below class heading
   - Right logo: 25mm width, 20mm height, positioned right-aligned (page width - 50mm), same Y as left logo
4. **Results table** (via jspdf-autotable)
   - **Columns:** Rank | Name | Gesamt (Sum)
   - **Header row:** Bold, filled background (gray), white text
   - **Body rows:** Regular weight, alternating row colors (striped theme)
   - **Theme:** striped (jspdf-autotable built-in theme)
   - **Font size:** 10pt body, 10pt bold header
   - **Cell alignment:** Rank (left), Name (left), Sum (left)
   - **Pagination:** Automatic (jspdf-autotable `pageBreak: 'auto'`)
   - **Header repeat:** Show on first page only (standard tables, not repeating headers)
5. **Page breaks**
   - Insert `doc.addPage()` before each new class (except first class)
   - No manual section breaks needed — jspdf-autotable handles multi-page tables

## Interaction Specs

### PDF Export Button Behavior
- **Trigger:** User clicks "PDF exportieren" button in Results view
- **Pre-flight checks:**
  - Verify at least one class has results (no export if zero classes with rankings)
  - Read current checkbox state (includeIncomplete)
- **Action flow:**
  1. Fetch current rankings via `computeClassRankings()`
  2. Filter by `includeIncomplete` checkbox state
  3. Fetch current settings (title, logo images) from Dexie `settings` table
  4. Call `generateResultsPdf()` pure function (from RESEARCH.md Pattern 1) with filtered rankings + settings
  5. Receive Blob (PDF bytes)
  6. Generate filename via ISO date: `Ergebnisse_2026-07-06.pdf`
  7. Trigger browser download via anchor element + blob URL
- **Error handling:**
  - QuotaExceededError (rare during PDF generation): "Speichern fehlgeschlagen" — user resets presets or clears incomplete shooters
  - Generic errors: "Speichern fehlgeschlagen" — user retries or contacts support

### Settings Form Behavior
- **Load:** On form mount, fetch singleton settings record (id: 1) from Dexie `settings` table; populate form fields with stored values
- **Image upload:**
  - User selects file via file picker (accept PNG/JPEG)
  - Pre-flight check: file.size > 200KB → error "Bild ist zu groß (max. 200KB)."
  - Downscale via Canvas (RESEARCH.md Pattern 2): 500px target width, 0.85 quality
  - Post-downscale check: blob size > 200KB (should not occur, but safeguard) → error "Upload fehlgeschlagen"
  - Show preview (img tag with src=dataUri, max-h-[80px])
- **Save (form submit):**
  - Validate title field (trim, allow empty)
  - Call `db.settings.put({ id: 1, title, logoLeftBlob, logoRightBlob })`
  - On success: form state updates via Dexie liveQuery reactivity (implicit via `$derived` of liveQuery result)
  - On error: display error message (14px red-600)
  - On QuotaExceededError specifically: show "Speicher voll — versuchen Sie kleinere Bilder."
- **Round-trip:** Preset export (via existing dexie-export-import, Phase 2) automatically includes settings table Blob fields; import restores images

- **Button minimum height:** 44px (iOS/Android touch target standard)
- **Checkbox clickable area:** 44px min-height (wraps label + input, full row clickable)
