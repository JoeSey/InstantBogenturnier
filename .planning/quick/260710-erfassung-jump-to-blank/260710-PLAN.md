<objective>
On the Erfassung (score entry) tab, when a tournament already has classes/archers configured and some scores already entered, the view should open showing the first round/passe that still contains a blank (unentered) arrow — instead of always defaulting to round 1 / passe 1. If no scores exist yet, or every score is already entered, keep the current default (round 1 / passe 1).

Purpose: Saves the trainer from manually clicking through already-completed rounds/passes every time they reopen or switch to Erfassung mid-tournament.
Output: A pure `findFirstIncompletePasse` helper (unit-tested) plus a one-time initialization effect in `ScoreEntry.svelte` that sets `selectedRound`/`selectedPasse` on first data load, without disturbing subsequent manual navigation.
</objective>
