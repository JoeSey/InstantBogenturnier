<objective>
Four independent v1.0 polish fixes on top of the completed milestone:

1. Desktop sidebar nav is 240px wide at the `xl` (>=1280px) breakpoint — shrink to 120px and adjust the main-content padding that compensates for it.
2. The Setup page is single-column at every viewport width — make it 2-column on `md` (>=768px) and above, matching the responsive conventions already used by Registration/ScoreEntry/Results (`max-w-[…]` + grid).
3. The "Runden und Passen" section (SetupRounds.svelte) requires an explicit "Speichern" click before a tournament can start, with no visual cue that it's required — remove the button and auto-save on every field change, matching the already-live-reactive "Schießplätze" field elsewhere on the same page.
4. `ScoreEntry.svelte` currently allows "Turnier abschließen" to become enabled with zero registered shooters (the underlying `areAllScoresEntered` is intentionally vacuously true for an empty shooter list), but finalizing with no score records never actually flips the persisted `isFinalized` state (which requires `scores.length > 0`) — so confirming looks like it does nothing. Fix: disable "Abschließen" and show a clear message when there are no registered shooters, instead of exposing a no-op finalize path.

Purpose: Ship the last round of low-risk UX/consistency fixes identified during final v1.0 review before this milestone is considered fully polished.
Output: Updated Sidebar/App shell CSS, a responsive Setup page grid, auto-saving SetupRounds with matching updated tests, and a guarded zero-shooter finalize path with matching updated tests. New/updated Playwright specs for the two visual changes (nav width, setup grid).
</objective>
