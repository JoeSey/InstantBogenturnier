Dexie v2 schema shape (src/lib/db/schema.ts), to be created by Task 1:
- `classes: '++id, name'` → `ClassRecord { id?: number; name: string; ageGroup?: string; bowType?: string; distance?: string }`
- `shootingLines: 'id'` (singleton row, always `id: 1`) → `ShootingLineConfig { id?: number; count: number }`
- `rounds: 'id'` (singleton row, always `id: 1`) → `RoundConfig { id?: number; arrowsPerPasse: number; passesPerRound: number; numberOfRounds: number; distance: string; presetId?: string }`
- `shooters: '++id, classId, lineAssignment'` → `ShooterRecord { id?: number; name: string; classId: number; lineAssignment?: number | null; flight?: 'A/B' | 'C/D' | null }`
- `presets: '++id, name'` → `PresetRecord { id?: number; name: string; classes: ClassRecord[]; shootingLineCount: number; roundsConfig: Omit<RoundConfig,'id'>; createdAt: Date }`

`src/lib/db/testHelpers.ts` exports `resetDb(): Promise<void>` — clears all 5 tables. Every later Dexie-touching test file (Plans 02, 03, 04) must call this in `beforeEach`.

`src/lib/utils/classNameGenerator.ts` exports:
- `getBowTypeAbbr(bowType: string): string`
- `generateClassName(ageGroup: string, bowType: string, distance: string): string`
- `autoSuffixOnCollision(baseName: string, tuple: { ageGroup?: string; bowType?: string; distance?: string }, existingClasses: ClassRecord[]): string`
