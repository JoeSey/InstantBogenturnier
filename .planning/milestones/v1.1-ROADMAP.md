**Post-ship cosmetic fixes (2026-07-07, fast-tracked outside the GSD phase workflow, directly on user request):**
- Results grid cards were squeezed into a fixed 3-column track even with fewer classes present, forcing ResultsTable's horizontal scroll unnecessarily — switched to an auto-fit grid (`minmax(480px, 1fr)`) so cards grow to use freed-up width.
- Added an always-visible thin scrollbar to ResultsTable's horizontal scroll wrapper — OS-level overlay scrollbars (e.g. macOS) stayed invisible until hover/scroll, making the (already-working) scroll affordance undiscoverable.
- Certificate layout redesigned per user feedback comparing against a real printed club certificate: both logos grouped into one centered, bigger block at the top (not pinned to margins), and the compact name/class/rank/score block replaced with a spread-out alternating connector/value line flow for a more "authentic" look.
- Fixed an old buglet surfaced during this pass: removing the explicit rounds/passes "Speichern" button (quick task 260706-9iv, v1.0) meant `db.rounds` only got written on an `onchange` event — a fresh install's default-selected WA-preset radio never fired one, so a new user navigating straight to Erfassung hit the "not configured" placeholder with no visible reason why. Now a one-shot direct read on mount detects a genuinely missing config and persists the visible default immediately.
