## Interaction Specs

### Bulk Certificate Export ("Urkunden erstellen" button)

**Trigger:** User clicks "Urkunden erstellen" button in Results view

**Pre-flight checks:**
- Verify at least one class has results (same as Phase 5 PDF export)
- Read current checkbox state (includeIncomplete) from Phase 5
- If zero shooters with results: show error "Keine Ergebnisse zum Exportieren" and return

**Action flow:**
1. Fetch current settings (title, logo images, certificate heading) from Dexie `settings` table (id: 1)
2. Fetch current rankings via `computeClassRankings()` (reuse Phase 4 logic)
3. Filter rankings by `includeIncomplete` checkbox state (reuse Phase 5 filter logic)
4. For each shooter in filtered rankings:
   - Call `buildCertPdf(shooter, className, settings)` to generate jsPDF document
   - Call `doc.output('blob')` to convert to Blob
5. Collect all Blobs in a JSZip instance (`new JSZip()`)
6. For each certificate Blob, add to ZIP with filename `Urkunde_<ShooterName>_<date>.pdf` (per D-08)
7. Call `zip.generateAsync({ type: 'blob', compression: 'DEFLATE' })` to produce final ZIP Blob
8. Generate ZIP filename: `Urkunden_YYYY-MM-DD.zip` (ISO date)
9. Trigger browser download via anchor element + blob URL (reuse Phase 5 WR-04 pattern: append to DOM before click)
10. Revoke blob URL and remove anchor from DOM

**Error handling:**
- JSZip creation fails: "Zip-Datei konnte nicht erstellt werden"
- jsPDF generation fails for a shooter: "Urkunden-Export fehlgeschlagen" (abort entire batch, do not partial-export)
- Generic/caught errors: "Urkunden-Export fehlgeschlagen" — user can retry

**Disable button if:**
- Zero shooters with results (same gate as Phase 5 PDF export)

### Per-Row Certificate Export (table action button)

**Trigger:** User clicks certificate action button (icon/text) in a single row of the results table

**Pre-flight checks:**
- Verify the shooter has a result in the selected class (they're in the table, so this is implicit)

**Action flow:**
1. Fetch current settings (title, logo images, certificate heading) from Dexie `settings` table
2. Get the shooter's data (name, class, rank, sum) from the clicked table row
3. Call `buildCertPdf(shooter, className, settings)` to generate jsPDF document
4. Call `doc.output('blob')` to convert to Blob
5. Generate filename: `Urkunde_<ShooterName>_<date>.pdf`
6. Trigger browser download via anchor element + blob URL (reuse Phase 5 WR-04 pattern)
7. Revoke blob URL and remove anchor from DOM

**Error handling:**
- Same as bulk export: "Urkunden-Export fehlgeschlagen" — user can retry
- Show error in the Results view error zone (above results table, shared with bulk export errors)

**Button always enabled** (if row is rendered, shooter has a result)

### Settings Form Behavior (Certificate Heading Field)

**Load:** On form mount, fetch singleton settings record (id: 1) from Dexie `settings` table; populate certificate heading field with stored value (default: "Urkunde" if not set)

**Edit:** User types freely into the certificate heading field — no validation, no character limits (text-only field)

**Save (form submit):**
- Validate certificate heading (trim whitespace, allow empty for default behavior)
- Call `db.settings.put({ id: 1, title, logoLeftBlob, logoRightBlob, certificateHeading })`
- On success: form state updates via Dexie liveQuery reactivity
- On error: display "Speichern fehlgeschlagen" in red text below form
- On QuotaExceededError: show "Speicher voll — versuchen Sie kleinere Bilder." (reuse Phase 5 error text)

**Sync:** When user navigates to Results view and clicks "Urkunden erstellen", the most-recently-saved certificate heading text is used (fetch from DB at click time, not at page load time)
