<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| File picker -> Canvas -> Dexie | User-selected image file (untrusted content, arbitrary bytes) crosses into client-side Canvas processing and then IndexedDB storage. |
| Form input -> Dexie -> (later) PDF text | Free-text title string crosses into persistent storage and will later be rendered as PDF text by Plan 02. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `imageDownscale.ts` file upload handler | mitigate | Validate `file.type.startsWith('image/')` before Canvas processing; malformed/non-image files throw before reaching `Image.onload`, surfaced as `errorUploadFailed`. |
| T-05-02 | Denial of Service | `SettingsForm.svelte` save handler | mitigate | Hard 200KB pre-flight size check before downscaling, plus post-downscale size assertion; catch `QuotaExceededError` explicitly and surface `errorQuotaExceeded` rather than an unhandled rejection. |
| T-05-03 | Information Disclosure | `dexie-export-import` (existing, unmodified) | accept | Single-device offline PWA with no secrets/auth; export is user-initiated only. No new risk introduced by adding Blob fields to an already-exported table. |
</threat_model>
